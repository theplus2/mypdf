# 목표
프로젝트의 빌드, 테스트, 배포 과정을 자동화하는 파이프라인을 구축하고 유지보수합니다.

# 지침
1. **워크플로우 생성:** 프로젝트 언어에 맞는 GitHub Actions 설정을 생성하세요. 가능한 최신 버전의 기능을 사용하되, 안정성이 검증된 방식을 택하세요.
2. **보안 관리:** API Key 등 비밀값이 코드에 노출되지 않게 검수하고, 설정 방법을 가이드하세요.
3. **단계별 검수:** [빌드 -> 테스트 -> 배포]가 순서대로 진행되게 하되, 이전 단계 실패 시 배포를 중단하세요.
4. **드라이런(사전 검증):** 실제 배포 전 가상 환경에서 빌드가 성공하는지 먼저 확인하세요.
5. **에러 자가 치유:** 실행 중 에러가 나면 로그를 분석해 스스로 수정안을 제시하세요.
