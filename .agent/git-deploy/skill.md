# 목표
안전하고 표준화된 방식으로 코드를 버전 관리하고, 실수를 방지하며 깃허브(또는 서버)에 업로드합니다.

# 지침 (Instructions)

## 1. 보안 검수 (가장 중요)
- 커밋하거나 푸시하기 전에 파일 내용 중 API Key, 비밀번호, 혹은 목사님의 개인적인 인증 정보가 포함되어 있는지 확인하세요.
- 만약 발견되면 사용자에게 "보안 위험이 있는 정보가 발견되었습니다. .env 파일로 분리할까요?"라고 먼저 물어보세요.

## 2. .gitignore 확인
- 프로젝트 언어(Flutter, Python 등)에 맞는 `.gitignore` 파일이 있는지 확인하고, 없다면 표준 템플릿으로 생성하세요.

## 3. 커밋 및 업로드 절차
- 사용자가 "업로드해줘"라고 하면 다음 순서를 따르세요:
  1. `git status`를 확인하여 변경된 내용을 요약하세요.
  2. 변경 사항에 어울리는 커밋 메시지를 'Conventional Commits' 규격(예: feat, fix, docs)에 맞춰 제안하세요.
  3. 원격 저장소(Remote URL)가 설정되어 있지 않다면 사용자에게 URL을 요청하세요.
  4. 사용자의 최종 확인을 받은 후 `git push origin main`을 실행하세요.

## 4. 배포 전 체크리스트 (Optional)
- 빌드 에러가 없는지 `flutter build`나 `npm run build` 등을 먼저 실행하여 확인하세요.
- 빌드가 성공했을 때만 푸시를 진행하세요.

# 제약 사항
- 사용자의 명확한 승인(예: "응, 올려줘") 없이는 절대 `push` 명령을 실행하지 마세요.
- 강제 푸시(`git push -f`)는 데이터 유실 위험이 있으므로 절대 사용하지 마세요.
