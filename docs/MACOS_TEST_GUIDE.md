# MacOS 테스트 가이드

이 문서는 Windows 환경에서 개발되어 GitHub Actions를 통해 빌드된 `MyPDFLibrary` 앱을 Mac에서 테스트하는 방법을 설명합니다.

## 1. 다운로드 및 설치

1. GitHub 저장소의 **Releases** 페이지 또는 **Actions** 탭에서 최신 빌드의 Artifact를 다운로드합니다.
2. 다운로드 받은 `MyPDFLibrary-Mac.zip` 파일의 압축을 풉니다.
3. `MyPDFLibrary.app` 파일이 나옵니다.

## 2. 앱 실행 방법 (중요)

Apple 개발자 계정으로 서명된 앱이 아니기 때문에(Ad-hoc 서명), 처음 실행 시 보안 경고가 나타날 수 있습니다.

### 방법 1: 우클릭으로 열기 (권장)
1. `MyPDFLibrary.app` 아이콘 위에서 마우스 오른쪽 버튼을 클릭합니다 (또는 Control + 클릭).
2. 메뉴에서 **열기(Open)**를 선택합니다.
3. "확인되지 않은 개발자가 배포한 앱..." 경고창이 뜨면 **열기** 버튼을 클릭합니다.
   - *참고: 그냥 더블 클릭하면 "열기" 버튼이 없는 경고창만 뜰 수 있습니다.*

### 방법 2: "앱이 손상되었습니다" 오류 해결
만약 "앱이 손상되었기 때문에 열 수 없습니다"라는 메시지가 뜨면, 이는 파일 자체가 깨진 것이 아니라 macOS의 보안 격리(Quarantine) 기능 때문입니다.

1. **Terminal(터미널)** 앱을 실행합니다 (Command + Space 후 'Terminal' 검색).
2. 다음 명령어를 입력하고 스페이스바를 한 번 누릅니다 (엔터 치지 마세요):
   ```bash
   xattr -cr 
   ```
3. Finder에서 `MyPDFLibrary.app`을 터미널 창으로 드래그 앤 드롭합니다. 그러면 경로가 자동으로 입력됩니다.
   ```bash
   xattr -cr /Path/To/Your/MyPDFLibrary.app
   ```
4. 엔터를 누릅니다. (아무 메시지도 안 나오면 성공입니다)
5. 다시 앱을 실행해 봅니다.

## 3. 문제 보고
여전히 실행되지 않거나 오류가 발생하면, 다음 정보를 포함하여 이슈를 등록해주세요.

- 실행 시 나타나는 정확한 오류 메시지
- macOS 버전
- (가능하다면) 터미널에서 앱 실행 시 출력되는 로그
  - 터미널에서 `MyPDFLibrary.app/Contents/MacOS/MyPDFLibrary` 를 직접 실행하면 로그를 볼 수 있습니다.
